Installation de PyMoDAQ

Cette notice explique comment installer PyMoDAQ sur Windows ainsi que le plugin Arduino, selon votre profil : utilisateur (TP) ou développeur.

Source : Cette notice s'appuie sur le guide officiel PyMoDAQ — pymodaq.cnrs.fr — Quick Start Windows.

1. Installation pour Utilisateur (TP)

Cette méthode est destinée aux élèves et enseignants qui souhaitent simplement utiliser PyMoDAQ lors des séances de TP, sans modifier le code source.

Étape 1 — Installer Miniforge

Miniforge est le gestionnaire d'environnements Python recommandé par PyMoDAQ. Téléchargez-le depuis le site officiel et exécutez l'installateur en conservant les paramètres par défaut.

Miniforge installe automatiquement mamba, Python et pip. Il remplace Anaconda/Miniconda et est plus rapide pour la création d'environnements.

Étape 2 — Ouvrir le Miniforge Prompt

Après installation, ouvrir le Miniforge Prompt depuis le menu Démarrer (pas le terminal Windows classique).

Étape 3 — Créer l'environnement Python

Créer un environnement dédié à PyMoDAQ avec Python 3.11 :

mamba create -n pymodaq_env python=3.11
# Confirmer avec y lorsque demandé, puis activer l'environnement :
mamba activate pymodaq_env
Attention : Choisissez l'avant-dernière version mineure de Python pour assurer la compatibilité avec tous les plugins PyMoDAQ. Évitez la toute dernière version.

Étape 4 — Installer PyMoDAQ

pip install pymodaq pyqt5

Étape 5 — Installer le Plugin Arduino

Une fois le plugin publié sur PyPI, l'installation se fait en une commande :

pip install pymodaq_plugins_dap_arduino
Note : La publication sur PyPI est en cours. En attendant, référez-vous à la méthode développeur ci-dessous pour installer le plugin depuis le dépôt GitLab.

Étape 6 — Vérifier l'installation

Pour vérifier que PyMoDAQ fonctionne correctement, lancez un module de test :

daq_move

Sélectionner Mock dans la liste déroulante et cliquer sur Initialization. Si la fenêtre s'ouvre sans erreur, l'installation est réussie.

2. Installation pour Développeur

Cette méthode est destinée aux développeurs qui souhaitent modifier le code du plugin ou travailler avec la version de développement de PyMoDAQ.

Étape 1 — Créer l'environnement Conda

Dans un Anaconda Prompt (ou Miniforge Prompt) :

# Créer l'environnement avec Python 3.12 :
conda create -n PYMODAQ_PROJET_2026 python=3.12
# Confirmer avec y, puis activer :
conda activate PYMODAQ_PROJET_2026
# Installer PyQt6 et PyMoDAQ :
pip install PyQt6
pip install pymodaq

Étape 2 — Installer le Plugin depuis GitLab

Ouvrir un PowerShell dans un dossier de votre choix (ex: D:\) et cloner le dépôt :

git clone https://XXX-CIEL:<TOKENPERSO>@gitlab.com/projet_pymodaq_2026/pymodaq-plugins-arduino

Puis dans l'Anaconda Prompt, installer le plugin en mode éditable (les modifications du code source sont prises en compte immédiatement) :

d:
cd pymodaq-plugins-arduino
pip install -e .

Étape 3 — (Optionnel) Installer PyMoDAQ en version dev

Si vous souhaitez travailler avec la version de développement de PyMoDAQ, clonez le dépôt GitHub dans un PowerShell :

git clone -b dev https://github.com/PyMoDAQ/PyMoDAQ

Puis dans l'Anaconda Prompt, installer les sous-paquets dans cet ordre :

pip install -e pymodaq_utils
pip install -e pymodaq_data
pip install -e pymodaq_gui

Étape 4 — (Optionnel) Outils supplémentaires

Pour le script de conversion des fichiers de données HDF5 vers CSV :

pip install h5py numpy

Étape 5 — (Optionnel) Recréer l'environnement depuis un export

Si un fichier d'export d'environnement Py26env.yml est disponible, vous pouvez recréer exactement le même environnement :

conda create --file Py26env.yml
