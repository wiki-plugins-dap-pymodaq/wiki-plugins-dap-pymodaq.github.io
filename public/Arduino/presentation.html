Dispositif Expérimental Arduino Nano ESP32

Banc d'essai thermique piloté par un Arduino Nano ESP32 via la librairie Telemetrix WiFi.

1. Introduction

Cette partie du wiki porte sur le développement d'un plugin PyMoDAQ utilisant un Arduino Nano ESP32 en tant que DAP (Dispositif d'Acquisition et de Pilotage). Le plugin permet de piloter à distance les broches du microcontroleur depuis un script Python, sans modifier le firmware pour chaque nouveau cas d'usage.

Ce projet est destiné aux séances de travaux pratiques des classes de physique de BTS CIEL du lycée Édouard Branly. L'objectif est de permettre le pilotage d'un ventilateur et d'un chauffage, ainsi que la lecture de la température via un capteur PT100.

2. Matériel et Dispositif Physique

Figure 1 : Vue du dispositif expérimental Arduino

Le montage matériel s'articule autour des éléments suivants :

  • Arduino Nano ESP32 : Contrôleur principal fonctionnant en 3,3 V. Il exécute le firmware Telemetrix et communique avec PyMoDAQ via WiFi (TCP/IP).
  • 2 Hacheurs XY-MOS : Modules MOSFET permettant de moduler la puissance envoyée au ventilateur et au chauffage via un signal PWM.
  • Module MAX31865 : Convertisseur dédié à la lecture des sondes PT100. Il communique avec l'ESP32 via l'interface SPI en logique 3,3 V.
  • Ventilateur : Assurant le refroidissement du dispositif, piloté en PWM sur GPIO17.
  • Sonde PT100 : Résistance en platine dont la valeur varie avec la température (≈ 100 Ω à 0 °C). Retenue pour sa précision et sa fiabilité dans les applications de laboratoire.

A. Supervision (L'Ordinateur)

L'ordinateur fait tourner un environnement virtuel Python qui héberge PyMoDAQ :

  • Plugin DAQ_Move_FanHeater : Envoie les consignes PWM (0–100 %) au ventilateur et au chauffage via Telemetrix.
  • Plugin DAQ_0DViewer_MAX31865 : Reçoit et affiche la température lue par le capteur PT100.
  • Librairie telemetrix_aio_esp32 : Assure la communication WiFi asynchrone avec l'ESP32.

B. Contrôle Matériel (L'Arduino Nano ESP32)

L'ESP32 exécute le firmware Telemetrix4ESP32WIFI, qui ouvre un serveur TCP sur le réseau local (port 31336 par défaut). Il reçoit les commandes du client Python et renvoie les données acquises.

  • Sorties PWM : GPIO17 (ventilateur) et GPIO18 (chauffage), via les hacheurs XY-MOS.
  • Interface SPI : Lecture du module MAX31865 pour acquérir la température de la sonde PT100.

4. Brochage du Dispositif

Figure 3 : Schéma de câblage du dispositif Arduino

Correspondance entre les composants et les broches de l'Arduino Nano ESP32 :

Module MAX31865 (interface SPI)

  • CLK → D13 (GPIO13)
  • SDO / MISO → D12 (GPIO12)
  • SDI / MOSI → D11 (GPIO11)
  • CS → D10 (GPIO10)
  • VIN → 3,3 V  |  GND → GND

Hacheurs XY-MOS (interface PWM)

  • Chauffage → GPIO18 (D9) — commande de 0 à 100 %
  • Ventilateur → GPIO17 (D8) — commande de 0 à 100 %
Attention : Telemetrix ESP32 utilise les numéros GPIO et non les labels sérigraphiés sur la carte (D8 ≠ GPIO8). Il faut utiliser 17 et 18 dans la configuration, et non 8 et 9.
