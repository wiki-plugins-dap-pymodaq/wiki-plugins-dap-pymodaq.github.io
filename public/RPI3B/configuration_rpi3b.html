Configuration de l'environnement Python (venv)

Cette page explique comment configurer l'environnement Python nécessaire pour faire tourner notre projet sur le Raspberry Pi. Pour éviter les conflits avec le système et garantir que tout le monde utilise les mêmes versions de bibliothèques, nous utilisons un environnement virtuel standard (venv) et un fichier de dépendances strict.

1. Préparation : Le fichier requirements.txt

Avant de commencer, assurez-vous d'avoir un fichier nommé requirements.txt à la racine de votre projet. Ce fichier contient la liste exacte des paquets et de leurs versions validées pour le projet.

Si vous ne l'avez pas encore, créez-le et collez ceci à l'intérieur :

Adafruit-Blinka==8.69.0
adafruit-circuitpython-busdevice==5.2.15
adafruit-circuitpython-connectionmanager==3.1.6
adafruit-circuitpython-emc2101==1.2.12
adafruit-circuitpython-register==1.11.1
adafruit-circuitpython-requests==4.1.15
adafruit-circuitpython-typing==1.12.3
Adafruit-PlatformDetect==3.86.0
Adafruit-PureIO==1.1.11
binho-host-adapter==0.1.6
board==1.0
pigpio==1.78
pyftdi==0.57.1
pyserial==3.5
pyusb==1.3.1
pyzmq==27.1.0
RPi.GPIO==0.7.1
smbus2==0.6.0
sysv_ipc==1.2.0
typing_extensions==4.15.0

2. Création de l'environnement virtuel

Ouvrez un terminal sur votre Raspberry Pi et placez-vous dans le dossier de votre projet :

cd /chemin/vers/votre/projet

Créez l'environnement virtuel nommé PY_ENV :

python -m venv PY_ENV

3. Activation de l'environnement (⚠️ Étape cruciale)

Avant d'installer quoi que ce soit ou de lancer vos scripts, vous devez toujours activer cet environnement.

Tapez la commande suivante :

source PY_ENV/bin/activate

(Indice visuel : votre ligne de commande doit maintenant commencer par (PY_ENV)).

4. Installation des dépendances

Maintenant que vous êtes dans l'environnement isolé, mettez à jour l'installateur pip, puis installez toutes les bibliothèques d'un seul coup grâce à notre fichier :

pip install --upgrade pip
pip install -r requirements.txt
Note : L'installation de certains paquets matériels sur Raspberry Pi peut prendre quelques minutes, c'est normal.

5. Vérification de l'installation

Pour vérifier que votre environnement est parfaitement configuré et identique à l'environnement de référence, tapez :

pip list

Vous devriez voir apparaître la liste exacte des paquets (avec les mêmes numéros de version) que ceux présents dans le fichier requirements.txt.

Pour vérifier que vous utilisez bien le Python de l'environnement virtuel et non celui du Raspberry Pi, tapez :

which python

Le résultat doit pointer vers le dossier PY_ENV (ex: /home/pi/.../PY_ENV/bin/python).
