Exécution & Services

Lancement du Script

Il existe deux méthodes pour lancer le script : soit vous le démarrez manuellement après chaque redémarrage du Raspberry Pi, soit vous configurez le système d'exploitation du Raspberry Pi pour qu'il le lance automatiquement à chaque mise sous tension.

1. Lancement manuel du Script

Pour exécuter le script manuellement, il est nécessaire d'activer l'environnement virtuel qui l'héberge.

# Activer l'environnement virtuel
source PY_ENV/bin/activate
# Lancer le script
python main.py
Avertissement : Vous devez démarrer le service pigpiod avant d'exécuter le script avec la commande python main.py.

2. Lancement automatique du Script (Service Systemd)

Pour lancer le script automatiquement, vous devez créer un fichier .service dans le répertoire /etc/systemd/system/ et y ajouter le code ci-dessous.

[Unit] 
# Nom affiché dans les logs et journalctl 
Description=Serveur ZeroMQ de pilotage thermique 
# Démarre ce service APRÈS que le réseau soit disponible 
# et APRÈS le démon pigpio (contrôle GPIO du Raspberry Pi) 
After=network.target pigpiod.service 

[Service] 
# Commande de lancement : utilise le Python de l'environnement 
# virtuel dédié au projet, et exécute le point d'entrée principal 
ExecStart=/home/admin/PYMODAQ_PROJET_2026/PY_ENV/bin/python /home/admin/PYMODAQ_PROJET_2026/main.py 
# Répertoire de travail du processus (important pour les imports 
# relatifs et la résolution des fichiers de config) 
WorkingDirectory=/home/admin/PYMODAQ_PROJET_2026 
# Redirige stdout/stderr vers le journal parent (journalctl) 
StandardOutput=inherit 
StandardError=inherit 
# Redémarre automatiquement le service en cas de crash
Restart=always 
# Utilisateur sous lequel tourne le processus (évite root) 
User=admin 

[Install] 
# Active le service au démarrage pour le niveau multi-utilisateur 
WantedBy=multi-user.target

Dans ce script, vous devez modifier les lignes ExecStart et WorkingDirectory pour spécifier votre chemin d'accès, ainsi que la ligne User pour indiquer le nom de votre session.

Après la création du fichier de service, exécutez les commandes suivantes pour appliquer la configuration et gérer le service :

# Recharger la configuration de systemd :
sudo systemctl daemon-reload
# Activer le lancement automatique au démarrage :
sudo systemctl enable pilotage.service
# Démarrer le service immédiatement (sans redémarrer le système) :
sudo systemctl start pilotage.service
# Vérifier le statut du service pour s'assurer qu'il n'y a pas d'erreurs :
sudo systemctl status pilotage.service

Si la commande sudo systemctl status pilotage.service ne signale aucune erreur, cela indique que votre script main.py est en cours d'exécution.
